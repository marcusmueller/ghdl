Python Interfaces¶

GHDL offers two Python interfaces and a language server protocol service. All this is provided from a pyGHDL packages with four sub-packages:

  • pyGHDL.cli - Command line interface (CLI) applications.

  • pyGHDL.dom - High-level API as document object model (DOM).

  • pyGHDL.libghdl - Low-level API to libghdl

  • pyGHDL.lsp - Language server protocol (LSP) implementation and service

pyGHDL.libghdl

pyGHDL.libghdl is a low-level API directly interacting with the shared library libghdl....so/libghdl....dll. This is a procedural and C-like interface.

It comes with some Python generators for easier iterating linked lists.

pyGHDL.dom

pyGHDL.dom is a high-level API offering a document object model (DOM). The underlying abstract VHDL language model is provided by pyVHDLModel. The DOM is using libghdl for file analysis and parsing.

pyGHDL.lsp

pyGHDL.lsp is language server protocol (LSP) written in Python. The implementation offers an HTTPS service that can be used e.g. by editors and IDEs supporting LSP.

Scripts and Applications¶

The pyVHDLParser package comes with an executables registered by pip in the search path.

  • VHDLParser is a wrapper for pyVHDLParser.CLI.VHDLParser:main.

ghdl-ls¶

VHDL Language Protocol Server

usage: ghdl-ls [-h] [--version] [--verbose] [--log-file LOG_FILE]
               [--trace-file TRACE_FILE] [--input INPUT] [--disp-config]

optional arguments¶

-h, --help¶

show this help message and exit

--version, -V¶

show program’s version number and exit

--verbose, -v¶

Show debug output

--log-file <log_file>¶

Redirect logs to the given file instead of stderr

--trace-file <trace_file>¶

Save RPC data to FILE.in and FILE.out (overrides GHDL_LS_TRACE)

--input <input>, -i <input>¶

Read request from file

--disp-config¶

Disp installation configuration and exit

              [--node-file NODE_FILE] [--template-file TEMPLATE_FILE]
              [--meta-basename META_BASENAME] [--kind-type KIND_TYPE]
              [--kind-prefix KIND_PREFIX]
              [--kind-range-prefix KIND_RANGE_PREFIX] [--node-type NODE_TYPE]
              [--keep-order]
